# 🧠 Comprehensive Mamba Models Performance Report (A to Z)

This report provides a detailed analysis of the two Mamba-based architectures implemented and trained for Glaucoma classification: **MambaOut** and **Vision Mamba (Vim)**.

---

## 1. Architecture Overview (A-C)
### A. Architecture Designs
- **MambaOut**: A "Mamba-like" vision model that removes the Recurrent/State-Space dependency (SSM) for vision tasks, relying instead on specific stacking blocks to achieve similar performance with lower complexity.
- **Vision Mamba (Vim)**: A pure State-Space Model (SSM) approach, applying bidirectional Mamba blocks to image patches.

### B. Backbones and Stems
- Both models use a **Conv2d-based stem** for initial feature extraction (Patch Embedding).
- **MambaOut** uses a deeper block structure (64 -> 128 channels) with GELU activation.
- **Vision Mamba** focuses on patch-wise scanning with BatchNorm and GELU.

### C. Complexity Comparison
- **MambaOut**: ~0.34 MB weights (nano-template).
- **Vision Mamba**: ~0.34 MB weights (nano-template).
- These are highly lightweight variants designed for real-time mobile/local screening.

## 2. Methodology and Training (D-L)
### D. Dataset (SMDG-19)
- Source: `bumbledeep/smdg-full-dataset` (Hugging Face).
- Subsets: Fundus images from multiple global sources.

### E. Training Parameters
- **Epochs**: 10
- **Batch Size**: 8-16
- **Optimizer**: Adam (MambaOut) / AdamW (Vision Mamba)
- **Learning Rate**: 1e-4

### F. Preprocessing
- Resize to **224x224**.
- ImageNet normalization (`[0.485, 0.456, 0.406]`).
- RGB conversion for legacy fundus scanned images.
